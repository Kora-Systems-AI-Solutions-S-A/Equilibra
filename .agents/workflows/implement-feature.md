Implemente a feature solicitada seguindo rigorosamente a arquitetura atual do projeto.

Regras:
- responda em português
- escreva todo o código em inglês
- escreva comentários no código em português
- preserve os padrões existentes
- não misture UI com regra de negócio
- UI não pode aceder APIs diretamente
- use store para coordenação de estado
- use services para acesso a dados
- mantenha o código simples, legível e sustentável
- não mova ficheiros sem necessidade explícita
- não duplique componentes ou lógica

Antes de escrever código:
1. identifique onde a feature deve ficar
2. confirme a responsabilidade de cada camada
3. proponha alterações pequenas e seguras

Na resposta:
- explique brevemente o plano
- depois mostre a implementação
- ao final, liste riscos, impactos e regressões possíveis
