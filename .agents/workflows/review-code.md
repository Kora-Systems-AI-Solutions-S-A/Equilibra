Revise o código enviado como um engenheiro fullstack sênior.

Faça uma revisão técnica estruturada, em português, priorizando problemas reais.

Avalie principalmente:
- arquitetura
- separação de responsabilidades
- legibilidade
- tipagem
- estado e reatividade
- duplicação
- acoplamento
- risco de regressão
- aderência à stack e aos padrões do projeto

Formato da resposta:
1. problemas críticos
2. problemas importantes
3. melhorias opcionais
4. conclusão objetiva

Se algo estiver correto, diga claramente.
Não invente problemas.
Priorize precisão sobre quantidade.
