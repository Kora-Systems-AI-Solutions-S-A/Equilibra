---
description: Serve para validar se algo está no lugar certo.
---

Faça uma análise arquitetural da implementação enviada com base na estrutura atual do projeto.

Verifique:
- se cada ficheiro está na camada correta
- se a responsabilidade está bem distribuída
- se UI, store, service e infraestrutura estão segregados corretamente
- se há violação de fronteiras
- se há lógica de negócio em componente visual
- se há acesso direto indevido a API
- se os imports seguem o padrão do projeto
- se a implementação está coerente com a arquitetura do Equilibra

Formato da resposta:
1. o que está correto
2. o que está fora do lugar
3. ajustes recomendados
4. decisão final: manter como está ou ajustar

Não reestruture tudo sem necessidade.
Prefira continuidade e consistência com o projeto atual.
